Question

How do we determine kkk, the number of clusters, given that we already know kkk means and DB SCAN?

Answer

If the dataset is 2 or 3 dimensional, then visually creating a scatter plot of the data and taking a guess at how many clusters could be there could work.

But more generally, we could run kkk means and DB SCAN on the data with an arbitrary kkk, and then for kkk means, we could look at the results and calculate, for each ppp in the dataset the distance d(p,c)d(p,c)d(p,c), where ccc is the centroid assigned to ppp, and add the distances up. Then we repeat this process for multiple values of kkk and choose the kkk for which the value of the distance sum, so to speak, is the least. This could work, perhaps.
(Small subsequent edit : We would need to enforce a lower limit for kkk otherwise trivially it will give us k=1k=1k=1.)

And perhaps we can do something similar for DBSCAN, we run DB SCAN for different values of kkk and ϵ\epsilonϵ and then see for which values of (k,ϵ)(k,\epsilon)(k,ϵ), the number of outliers and the distance sum of the outliers from the nearest cluster is minimised.
